What is GridSim?

A slave station simulator for IEC 60870-5-104 protocol, designed for substation automation testing. It simulates RTUs and bay-level devices for SCADA system development and integration testing.

  • Multi-instance — Up to 1000 parallel instances
  • Full protocol — AI/DI/PI/DO/AO point types
  • Web UI — Vue 3 management interface
  • MCP Server — AI Agent programmatic control
// Traditional Mode (single instance)
./gridsim -p 2404 -c point.xlsx

// Serve Mode (multi-instance)
./gridsim serve \
  --http :8989 \
  --config-dir ./config \
  --log-dir ./logs

Point Types (测点类型)

Type 中文 Function IEC 104 ID Data
AI 遥测 YC Analog monitoring M_ME_NC_1 (13) float32
DI 遥信 YX Digital monitoring M_SP_NA_1 (1) bool 0/1
PI 遥脉 YM Pulse counting M_IT_NA_1 (15) int32
DO 遥控 Remote control C_SC_NA_1 (45) Control → DI
AO 遥调 Remote regulation C_SE_NC_1 (48) Control → AI

Supports general interrogation (C_IC_NA_1), energy call (C_CI_NA_1), spontaneous upload (COT=3), and QDS quality descriptors.

Auto-Change Strategies (1/2)

11 built-in strategies, each running in independent goroutines with configurable periods (≥100ms).

increment
Step += N per cycle, loops at max
random
Random value within [min, max]
csv
Time-series playback from CSV file
max
Maximum of multiple IOA points
min
Minimum of multiple IOA points
soc
Battery SOC via power integration
energy
Cumulative energy calculation
aofollow
Follow AO remote regulation value
apiupdate
HTTP API write only, engine skips
manual
Manual value set via API
custom ⭐
Custom formula with 2-50 point refs

Custom Formula Strategy v2.2.0

Build arithmetic formulas using a button-style editor with real-time preview.

  • Operators: + - * / ( )
  • 2~50 associated points via {0} {1}...
  • Recursive descent parser with bracket priority
  • Division-by-zero protection
  • Configurable period (≥100ms)
  • CSV import/export support (strategy code 11)
// Formula example:
// Temperature = (P1 + P2) / 2 * P3
"{0}+{1})/2*{2}"

// CSV format:
// A: IOA | B: Formula | C: Period(ms)
16385, ({0}+{1})/2, 1000
16386, {0}*{1}/{2}, 500

MCP Server — AI Agent Integration

Model Context Protocol server enables AI Agents to programmatically control the simulator for automated testing.

Instance Tools

  • list_instances — List all instances
  • create_instance — Create new instance
  • start_instance — Start instance
  • stop_instance — Stop instance
  • restart_instance — Restart instance
  • get_server_status — Global status

Data Tools

  • write_points — Batch point writes
  • read_points — Batch point reads
  • config_auto_change — Set strategy
  • export_auto_changes — Export CSV
  • import_auto_changes — Import CSV
  • upload_csv — Upload replay file
# Start MCP server
./bin/mcp-server -simulator http://localhost:8989 -mode both
