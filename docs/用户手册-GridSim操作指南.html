GridSim 用户操作手册

GridSim 是一个生产级 IEC104 从站模拟器，支持 Modbus TCP 协议和微电网仿真。本文档帮助用户快速上手并掌握全部功能。

💡 文档适用版本
本文档面向 GridSim v2.5.3。如使用旧版本，部分功能可能不可用。

运行模式

模式命令说明
传统模式gridsim -p 2404 -c point.xlsx单进程 = 单端口 + 单客户端，纯 CLI
服务模式gridsim serve --http :8989多实例生命周期管理 + Vue 3 Web 界面（推荐）

启动模拟器

方式一：下载压缩包（推荐）

  1. 解压tar xzf gridsim-v2.5.3.tar.gz && cd gridsim-v2.5.3
  2. 启动./start.sh
  3. 访问http://localhost:8989

方式二：源码构建

# 1. 构建前端
cd web && npm install && npm run build && cd ..

# 2. 构建后端
go build -o bin/gridsim ./cmd/gridsim/

# 3. 启动服务
./bin/gridsim serve --http :8989 --config-dir ./config --log-dir ./logs

# 4. 浏览器访问
open http://localhost:8989

传统模式（单实例）

go build -o bin/gridsim ./cmd/gridsim/
./bin/gridsim -p 2404 -c samples/point.xlsx -H :8080 -l info

DetailPage/instance/:id实例详情。测点实时值、置数、自动变化策略配置、CSV 多测点回放

💡 登录
首次启动，使用预设账号 admin / admin 登录。
MonitorPage — 自动刷新显示所有实例运行状态、客户端连接和运行统计。

ConfigPage — 实例管理

在 ConfigPage 中可以创建新的模拟器实例，每个实例对应一个独立的协议端口。 创建时需要指定实例名称、协议类型（IEC104 / Modbus TCP）、端口号和点表文件。 实例上限为 1000。页面顶部显示全局状态统计。

ConfigPage — 实例列表，显示已配置/1000、运行中、已停止计数。支持创建、编辑、启动、停止、删除实例。
DetailPage — 测点实时值、置数、自动变化策略配置、CSV 多测点回放，一站式操作。

DetailPage — 实例详情

实例详情页是日常操作的核心页面。进入后可以看到该实例所有测点的实时值列表。 支持以下操作：

  • 测点轮询：100ms~1000ms 可调高频刷新
  • 置数操作：AI 输入数值、DI 开关（ON/OFF）、PI 输入整数
  • 自动变化配置：为每个测点配置 11 种策略之一
  • 批量配置：勾选多个测点，一次性应用相同策略
  • CSV 多测点回放：独立卡片，上传 CSV 文件映射多列到不同测点
  • 导出/导入：测点 CSV 导出、自动变化配置 JSON 导出/导入
TrendPage — ECharts 多线趋势对比，缩放拖拽、十字准线 tooltip、图例显隐。

TrendPage — 趋势对比

选择多个测点，实时绘制趋势曲线。基于测点真实时间戳渲染。 支持缩放拖拽、十字准线 tooltip、图例显隐。时间范围 5m/15m/30m/1h/2h。

IEC104 协议操作

测点类型映射

PointType中文功能ASDU TypeID数据
AI遥测 YC模拟量监视M_ME_NC_1 (13)float32，值 = base × efficient
DI遥信 YX数字量监视M_SP_NA_1 (1)bool 0/1
PI遥脉 YM脉冲计数M_IT_NA_1 (15)int32
DO遥控远程控制C_SC_NA_1 (45)接收控制，更新 DI 点
AO遥调远程调节C_SE_NC_1 (48)接收控制，更新 AI 点

协议行为说明

  • 总召唤：客户端发送 C_IC_NA_1 (COT=6)，模拟器按 AI/DI/PI 分组返回所有测点值，最后发送 ACT_TERM (COT=10)
  • 变化上送：任何值变更（HTTP API 置数、自动变化引擎计算、AO/DO 控制）自动触发 COT=3 变化上送
  • 单客户端：每个实例同一时刻只接受一个 IEC104 客户端连接
  • 品质描述 QDS：支持 invalid / not_topical / substituted / overflow / blocked 五个品质位
⚠️ 注意事项
  • 变化周期最小值 100ms
  • AO/DO 不支持置数和自动变化
  • DI 置数只能设为 0 或 1，界面以 ON/OFF 开关展示

测点操作

HTTP API 置数

# 遥测值（AI）
curl -X PUT http://localhost:8989/api/v1/instances/{id}/points/16385 \
  -H 'Content-Type: application/json' \
  -d '{"value": 235.5}'

# 遥信值（DI）
curl -X PUT http://localhost:8989/api/v1/instances/{id}/points/5 \
  -H 'Content-Type: application/json' \
  -d '{"bool_value": true}'

# 批量读取
curl http://localhost:8989/api/v1/instances/{id}/points/batch?ioas=16385,16386,16387

QDS 品质描述更新

curl -X PUT http://localhost:8989/api/points/16385/qds \
  -H 'Content-Type: application/json' \
  -d '{"invalid": true, "blocked": true}'
⚠️ 重要说明
  • 置数值与实时数据隔离存储，不随轮询刷新覆盖
  • 置数成功后以绿色 toast 提示，无需二次确认
  • 配置了"接口更新"或"手动"策略的点，引擎不自动计算
  • 品质位变化不会触发变化上送，但会在下次总召时携带

自动变化策略

GridSim 提供 11 种自动变化策略，每个实例的每个测点可以独立配置。后台独立 goroutine 调度，实例启停自动管理。

#策略策略代号适用场景
1递增increment模拟缓慢上升的遥测量
2随机random模拟噪声或波动信号
3CSV 回放csv回放真实录波数据
4MAXmax联锁逻辑，多 IOA 取最大
5MINmin联锁逻辑，多 IOA 取最小
6SOC 计算soc基于功率积分计算荷电状态
7电量统计energy功率积分累计电量
8AO 关联aofollow跟随 AO 遥控值变化（事件驱动）
9接口更新apiupdate仅允许 HTTP API 写入
10手动manual不自动计算，需 API 置数
11自定义公式custom四则运算公式，2~50 关联测点

配置示例（递增策略）

curl -X PUT http://localhost:8989/api/v1/instances/{id}/points/auto-change/16385 \
  -H 'Content-Type: application/json' \
  -d '{
    "strategy": "increment",
    "enabled": true,
    "params": {
      "start_value": 0,
      "step": 1,
      "period_ms": 1000,
      "max_value": 100
    }
  }'

批量配置

curl -X PUT http://localhost:8989/api/v1/instances/{id}/points/auto-change/batch \
  -H 'Content-Type: application/json' \
  -d '{
    "ioas": [16385, 16386, 16387],
    "config": {
      "strategy": "increment",
      "enabled": true,
      "params": {"start_value": 0, "step": 1, "period_ms": 1000, "max_value": 100}
    }
  }'

导出/导入

# 导出自动变化配置
curl -H 'Accept: application/json' \
  http://localhost:8989/api/v1/instances/{id}/points/auto-change/export

# 导入自动变化配置
curl -X POST http://localhost:8989/api/v1/instances/{id}/points/auto-change/import \
  -H 'Content-Type: application/json' \
  -d '{"autoChanges": {"16385": {"strategy": "increment", ...}}}'

CSV 多测点同步回放

自 v2.5.0 起支持，在实例详情页有独立"CSV 多测点回放"卡片。

CSV 文件格式

time,母线电压,线路电流,有功功率
0,220.0,5.2,1144.0
1000,221.5,5.3,1173.9
2000,219.8,5.1,1120.9

操作流程

  1. 进入实例详情页，找到 CSV 回放卡片
  2. 上传 CSV 文件，或从已有文件下拉选择（共享目录 + 实例私有目录）
  3. 列映射：自动按列名匹配测点 IOA，回退按序号分配
  4. 选择时间模式：相对时间（ms/s）或绝对时间（hh:mm:ss）
  5. 启动回放：多个测点共享同一时间基准，严格同步

时间模式说明

模式说明
相对时间第 1 列为相对时间（ms 或 s），基于 elapsed time 控制播放节奏
绝对时间时间格式 hh:mm:ss，按系统时钟定位当前行
✅ 向后兼容
单测点 CSV 模式完全不变，多测点回放为新增独立功能。

使用示例

# 批量写入测点
write_points(
    instance_id="inst-001",
    points=[
        {"ioa": 16385, "value": 235.5},
        {"ioa": 16386, "bool_value": True}
    ]
)

# 配置 CSV 多测点回放
config_csv_replay(
    instance_id="a1b2c3d4e5f6",
    csv_file="replay_data.csv",
    time_format="relative",
    time_unit="ms",
    mappings=[
        {"column": 1, "ioa": 16385},
        {"column": 2, "ioa": 16386},
    ]
)
⚠️ MCP 故障排查
  • 连接失败：确认模拟器已启动，-simulator 地址正确
  • 测点写入失败：确认 IOA 存在于点表中，实例已启动
  • MCP 无响应：检查模拟器日志，确认点表已加载

微电网仿真

自 v2.5 起支持，提供设备管理和实时仿真能力。

支持的设备类型

类型标识说明
光伏pv发电设备，额定功率 + 效率
储能battery电池设备，容量 / SOC / 功率
负荷load用电设备，额定功率 + 功率因数
充电桩charger充电设备，额定功率 + 效率

引擎执行顺序（tick 循环）

  1. PV 功率计算（AO setpoint / 保持当前值）
  2. 负荷/充电桩功率（保持 store 值）
  3. 储能功率（AO setpoint / 保持）+ SOC 更新
  4. 功率平衡 GRID = Load + Charger + Battery − PV
  5. 同步 store + 公式求值
  6. 快照记录

控制模式

模式说明
远方 (remote)跟随 AO 遥控值
本地 (local)由策略驱动（自定义公式等）
💡 功率约定
PV 始终 ≥0（发电）。Battery：+ 充电，− 放电。GRID_P：+ 从电网用电，− 向电网送电。

常见问题

模拟器启动报错

检查端口是否被占用、点表文件是否存在、配置文件目录权限。

Web 界面无法访问

确认 --http 参数正确，防火墙未屏蔽端口。检查日志中是否有启动成功记录。

MCP 连接失败

Error: dial tcp connection refused

检查：模拟器是否已启动、-simulator 地址是否正确、HTTP 端口是否可达。

IEC104 客户端无法连接

检查实例是否已启动（Web 界面中状态应为 running）。每个实例只接受一个客户端连接。

测点值不变化

确认自动变化策略已启用（enabled=true）。检查策略参数是否正确。CSV 回放检查文件是否上传、列映射是否正确。

置数失败

确认 IOA 存在于点表中。配置了"接口更新"或"手动"策略的点，只能通过 API 写入。

⚠️ 安全提醒
  • JWT token 存储在 localStorage（已知 XSS 风险），生产环境建议配置 HTTPS + CSP 头
  • HTTP 服务缺少超时配置，生产环境建议使用反向代理（nginx/Caddy）
  • API 无速率限制，建议生产环境添加
