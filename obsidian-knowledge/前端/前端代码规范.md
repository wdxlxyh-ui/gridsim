# 前端代码规范

## 表格行组件复用

> [!warning] 关键约束
> `el-table` 复用 DOM 行，切换 tab 或更新数据时组件状态可能残留。

**解决方案**：为有状态组件添加 `:key="row.ioa"`。

```vue
<el-switch :model-value="!!setValues[row.ioa]" :key="row.ioa" />
```

## 置数值隔离

- `setValues[ioa]` 存储用户手动输入，与后端 `row.value` 完全隔离
- `fetchPoints` 初始化时用 `if (p.ioa in setValues) return` 保护
- `displayValue()` 只读后端数据

## 事件处理

| 场景 | 推荐写法 |
|------|----------|
| 输入框值变更 | `@update:model-value="(v) => { setValues[row.ioa] = v }"` |
| Enter 确认 | `@keydown.enter.prevent="() => doSetValue(row)"` |
| 开关切换 | `@change="(val: boolean) => doSetValue(row, val ? 1 : 0)"` |

## 禁用逻辑

```typescript
// ✅ AI/PI/DI 可编辑，无论策略如何
// ✅ AO/DO — 无输入框，显示"—""
// ❌ 禁止：根据 autoStrategies 禁用置数框
```

## API 调用

```typescript
// ✅ 正确
try {
  await setPointValue(instanceId.value, ioa, body)
  ElMessage.success('操作成功')
} catch (e: any) {
  ElMessage.error('操作失败: ' + (e?.response?.data?.error || e.message))
}

// ❌ 禁止：.then()/.catch() 链式
// ❌ 禁止：空的 catch
```
