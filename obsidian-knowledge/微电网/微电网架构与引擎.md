# 微电网架构与引擎

## 模块结构

```
internal/microgrid/
  model.go             数据模型（Topology/Device/FormulaRule）
  engine.go            仿真引擎（tick/功率平衡/SOC/IOA索引）
  pointmap.go          IEC104 测点展开
  handler.go           REST API（6组端点）
  history.go           历史快照缓冲区
  formula_test.go      单元测试（21个）
```

## 引擎执行流程（tick 循环）

每一 tick 周期依次执行 6 步：

```
tick() 每一周期:
  1. PV 功率计算（AO setpoint / 保持当前值）
  2. Load/Charger 功率（保持 store 值）
  3. Battery 功率（AO setpoint / 保持当前值）
     3.5 SOC 更新（ΔSOC = power × dt / capacity × 100）
  4. 功率平衡（GRID = Load + Charger + Battery - PV）
  5. 同步 store（_Power/_SOC/_SwStatus/_SwCtrl/_Status/GRID_P）
     5.5 公式评估（自动 GRID 公式 + 用户自定义公式）
  6. 快照记录
```

## Engine 核心结构

```go
type Engine struct {
    topology *Topology
    store    *library.Store

    soc       map[string]float64  // device_id → current SOC
    pvPower   map[string]float64  // device_id → PV power
    loadPower map[string]float64  // device_id → load/charger power
    batPower  map[string]float64  // device_id → battery power
    gridPower float64             // 并网点功率

    pointIOA map[string]uint32   // name → IOA 索引
    history  *HistoryBuffer      // 历史快照
}
```

## 功率正负约定

| 量 | + 含义 | - 含义 |
|----|--------|--------|
| PV | 发电 | 无（始终 ≥0） |
| Battery | 充电 | 放电 |
| Load / Charger | 用电 | — |
| GRID_P | 从电网用电 | 向电网送电 |

## 控制模式

| 模式 | 说明 |
|------|------|
| `remote` (远方) | 跟随 AO 遥控值 |
| `local` (本地) | 由策略驱动（自定义公式等） |

## 数据模型

### Topology

| 字段 | 类型 | 说明 |
|------|------|------|
| `devices` | Device[] | 设备列表 |
| `switches` | DeviceSwitch[] | 开关列表 |
| `formulas` | FormulaRule[] | 自定义公式 |
| `control_mode` | string | remote / local |
| `tick_ms` | int | 仿真周期(ms) |

### DeviceParams（各设备专属参数）

**PV**: `rated_power_kw`, `efficiency`
**Battery**: `capacity_kwh`, `rated_power_kw_b`, `init_soc`, `soc_min`, `soc_max`, `eff`
**Load**: `load_rated_kw`, `power_factor`
**Charger**: `charger_rated_kw`, `charger_eff`
