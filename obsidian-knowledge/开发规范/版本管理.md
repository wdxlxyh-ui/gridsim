# 版本管理

## 版本号规则

| 部分 | 说明 |
|------|------|
| 主版本号 | 不兼容 API 修改 |
| 次版本号 | 新功能发布 |
| 修订号 | Bug 修复 |
| 开发版本 | `-dev` 后缀 |

## 同步位置

版本号在以下位置保持一致：

1. `Makefile` — `VERSION := 2.5.3`
2. `web/package.json` — `"version": "2.5.3"`
3. `cmd/gridsim/main.go` — `-ldflags="-X main.version=2.5.3"`
4. Git Tag — `v2.5.3`

## 分支策略

| 分支 | 用途 |
|------|------|
| `main` | 稳定版本，仅接受 merge request |
| `develop` | 开发分支，新功能/修复在此完成 |

### 工作流程

1. 在 `develop` 分支开发并测试
2. 确认无误后 merge 到 `main`
3. 打 tag 并打包发行
