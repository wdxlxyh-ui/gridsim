# 设计优化报告 (v2.3.0)

## 前端优化

### JWT 解析增强
解析失败时自动清除无效 token 并重定向到登录页。

### 401 重定向优化
改用 `history.replaceState` 保持页面状态。

### TrendPage 性能
- 串行请求 → `Promise.all` 并行
- 数据获取时间减少约 50%

## 后端优化

### 中间件层
新增 `pkg/middleware/recovery.go`：捕获 panic、记录请求、统一错误响应。

### 日志级别调整
自动变化上送日志从 Info 降至 Debug，减少生产日志噪音。

### 统一错误响应

```json
{
  "error": "无效的参数",
  "code": 400,
  "details": {
    "field": "port",
    "issue": "must be between 1-65535"
  }
}
```

## 构建优化

- 版本号自动从 git tag 读取
- 生成 VERSION 文件
- 自动创建默认 `users.json`

## 性能基准

| 操作 | 优化前 | 优化后 | 改善 |
|------|--------|--------|------|
| TrendPage 首次加载（20 点） | ~800ms | ~400ms | 50% |
| 自动变化日志输出 | ~20 行 INFO/s | 0 行 INFO/s | 显著降低 |

## 已知限制

1. JWT 存储在 localStorage（XSS 风险）
2. 缺少单元测试
3. 未实施 API 速率限制
