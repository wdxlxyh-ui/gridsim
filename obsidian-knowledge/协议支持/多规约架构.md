# 多规约架构

## 设计原则

1. **协议抽象层** — 将"规约"抽象为可插拔的 Protocol 接口
2. **数据层复用** — `library.Store` 保持不变，所有规约共享同一数据源
3. **配置向后兼容** — `protocol` 字段默认 `"iec104"`，旧配置无需迁移
4. **点表向后兼容** — Modbus 专属列可选解析

## Protocol 接口

```go
type Protocol interface {
    Name() string
    Start() error
    Stop()
    ClientConnected() bool
    ClientAddr() string
    Stats() (interrog, control, spont int64)
    Uptime() int64
    Publish(point *config.Point)
    SetStore(store *library.Store)
}
```

## InstanceConfig 扩展

```go
type InstanceConfig struct {
    // ... 原有字段
    Protocol     string              `json:"protocol,omitempty"`
    ModbusConfig *ModbusInstanceConfig `json:"modbus_config,omitempty"`
}
```

## 目录结构

```
pkg/protocol/
  protocol.go        Protocol 接口定义
  factory.go         协议工厂（根据字符串创建实例）
  iec104_wrapper.go  IEC104 适配器
  modbus/
    tcp_server.go    Modbus TCP 服务端
    handler.go       功能码分发处理
    converter.go     数据类型转换
```

## 向后兼容性

| 场景 | 行为 |
|------|------|
| 旧 JSON 配置 | protocol 为空 → 默认 iec104 |
| 旧 Excel 点表 | 无 H/I/J 列 → 零值回退 |
| 旧版前端 | 不发送 protocol → 后端默认 iec104 |
