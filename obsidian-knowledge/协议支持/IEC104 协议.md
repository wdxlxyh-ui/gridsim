# IEC104 协议

## 测点类型映射

| PointType | 中文 | 功能 | ASDU TypeID | 数据类型 |
|-----------|------|------|-------------|----------|
| AI | 遥测 YC | 模拟量监视 | M_ME_NC_1 (13) | float32, value = base × efficient |
| DI | 遥信 YX | 数字量监视 | M_SP_NA_1 (1) | bool 0/1 |
| PI | 遥脉 YM | 脉冲计数 | M_IT_NA_1 (15) | int32 |
| DO | 遥控 | 远程控制 | C_SC_NA_1 (45) | 接收控制，更新 DI 点 |
| AO | 遥调 | 远程调节 | C_SE_NC_1 (48) | 接收控制，更新 AI 点 |

## 总召唤流程

```
收到 C_IC_NA_1 (COT=6 activation)
  ↓
Store.ByType 分组
  ├── AI 组 → M_ME_NC_1 ASDU (COT=7) → Send
  ├── DI 组 → M_SP_NA_1 ASDU (COT=7) → Send
  └── PI 组 → M_IT_NA_1 ASDU (COT=7) → Send
  ↓
发送 C_IC_NA_1 ACT_TERM (COT=10) → 总召结束
```

## 变化上送流程

```
触发条件:
  ├── HTTP PUT /api/points/{ioa} 修改值
  ├── HTTP POST /api/points/batch 批量修改
  ├── AO/DO 控制命令到达
  └── 内部 SetValue() 调用
  ↓
Store.SetValue(ioa, newValue)
  ├── 加锁写入
  ├── Point.Changed = true
  ├── 更新 Timestamp
  └── 通过 channel/回调 通知 Publisher
  ↓
Publisher 立即构造 ASDU (COT=3 spontaneous)
  └── 通过当前连接发送
```

## AO/DO 控制流程

```
接收 C_SC_NA_1 (DO) 或 C_SE_NC_1 (AO)
  ↓
解析 ASDU，提取 IOA + 新值
  ↓
验证: IOA 存在 && point-type 匹配
  ↓
匹配:
  ├── Store.SetValue(ioa, receivedValue)
  ├── 发送 ACT_CON (COT=7)
  ├── 发送 ACT_TERM (COT=10)
  └── 变化上送新值
  ↓
不匹配:
  └── ACT_CON + 负确认 (COT=44)
```

## 单客户端管理

```
状态: waiting / connected

连接到达 → 状态 = connected?
  ├── 是 → 拒绝
  └── 否 → 接受，记录 remoteAddr

客户端断开 → 状态 = waiting
```

## 电度召唤

支持 C_CI_NA_1 电度召唤，与总召流程类似。
