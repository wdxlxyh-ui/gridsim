// overview

这是什么？解决什么问题？

IEC104 从站模拟器，用于变电站自动化测试。模拟 RTU 及间隔层设备，适用于 SCADA 系统开发和集成测试。

✦ 替代真实硬件，降低测试成本
✦ 支持最多 1000 个实例并行运行
✦ 完整 IEC 60870-5-104 协议栈
✦ AI Agent 可通过 MCP 程序化控制
quick-start.sh
# 解压即用
tar xzf iec104-sim-v2.2.0.tar.gz
cd iec104-sim-v2.2.0
./start.sh
# → http://localhost:8989
这个项目的核心价值是替代真实硬件设备。在 SCADA 系统开发中，我们不需要等待真实的 RTU 设备到位，用模拟器就可以完成大部分集成测试工作。而且支持最多 1000 个实例并行，这在真实硬件环境下是不可能做到的。

// point-types

五种测点类型

类型中文功能IEC 104 标识数据格式
AI遥测 YC模拟量监视M_ME_NC_1 (13)float32
DI遥信 YX数字量监视M_SP_NA_1 (1)bool 0/1
PI遥脉 YM脉冲计数M_IT_NA_1 (15)int32
DO遥控远程控制C_SC_NA_1 (45)控制 → 更新 DI
AO遥调远程调节C_SE_NC_1 (48)控制 → 更新 AI

支持总召唤 C_IC_NA_1、电度召唤 C_CI_NA_1、变化上送 COT=3、品质描述 QDS

AI 是遥测，比如电压电流温度这些连续量。DI 是遥信，开关状态、断路器位置这些 0/1 信号。PI 是遥脉，电表累计读数。DO 和 AO 是反向的——外部系统发控制命令过来，模拟器接收后更新对应的 DI 或 AI 点。

// auto-change-strategies

11 种自动变化策略

increment每周期 += 步长，达最大值后回起始
random在 [min, max] 范围内随机取值
csv按 CSV 时间序列文件回放真实数据
max取多个 IOA 测点的最大值
min取多个 IOA 测点的最小值
soc基于功率积分计算电池 SOC
energy基于功率积分计算累计电量
aofollow跟随 AO 遥控值同步更新
apiupdate仅允许 HTTP API 写入
manual引擎不计算，需 API 手动置数
custom ⭐自定义公式，2~50 个关联测点
每个策略都是独立 goroutine 运行，周期可配置，最小 100ms。increment 模拟缓慢上升的遥测量，random 模拟噪声波动，csv 回放真实录波数据，SOC 和 energy 用于储能系统仿真。custom 是 v2.2.0 新增的重磅功能，下一页细讲。

// custom-formula · v2.2.0

自定义公式策略

通过按钮式编辑器构造四则运算公式，实时计算关联测点值。

✦ 运算符：+ - * / ( )
✦ 2~50 个关联测点，{0} {1} ... 占位符
✦ 递归下降解析器，支持括号优先级
✦ 除零保护，计算周期 ≥ 100ms
✦ CSV 导入导出（策略代码 11）
formula-example
// 公式：(P1 + P2) / 2 × P3
"({0}+{1})/2*{2}"

// CSV 格式：
// A:IOA  | B:公式        | C:周期(ms)
16385, ({0}+{1})/2, 1000
16386, {0}*{1}/{2},  500
这个功能的核心价值是：不需要写代码，通过 UI 就能构造复杂的测点联动关系。比如温度 = (传感器A + 传感器B) / 2 × 补偿系数。前端有按钮式编辑器，点选测点、点选运算符，公式实时预览。后端用递归下降解析器，支持括号优先级和除零保护。

// mcp-server

MCP — AI Agent 程序化控制

实例管理工具

  • list_instances — 列出所有实例
  • create_instance — 创建新实例
  • start_instance — 启动实例
  • stop_instance — 停止实例
  • restart_instance — 重启实例
  • get_server_status — 全局状态

数据接口工具

  • write_points — 批量写入测点 ⭐
  • read_points — 批量读取测点
  • config_auto_change — 配置策略
  • export_auto_changes — 导出 CSV
  • import_auto_changes — 导入 CSV
  • upload_csv — 上传回放文件
mcp-server
# 启动 MCP 服务器
./bin/mcp-server -simulator http://localhost:8989 -mode both
MCP 是 Model Context Protocol，让 AI Agent 像调用函数一样控制模拟器。write_points 是核心工具——一次写入多个测点，模拟真实设备同一时刻上报数据。这对自动化测试至关重要，因为真实场景下多个测点是同时变化的。
